General

Directory where .strm and .nfo files are written. Must be inside a Jellyfin library.
Externally accessible URL of this Jellyfin instance used in .strm resolver links (e.g. https://jellyfin.example.com). Must be reachable by remote clients.
How long a resolved CDN URL is kept in memory before being re-fetched from YouTube.
Used when Enhanced playback is disabled or unavailable. Compatibility first prefers simpler 720p-friendly streams, Balanced aims for a good middle ground, and Highest single-stream quality asks YouTube for the best single stream it can provide.
Maximum number of videos to sync per channel or playlist. Set to 0 for no limit (not recommended for large channels).

Enhanced Playback

Simple mode keeps setup light and works without ffmpeg. Enhanced mode uses ffmpeg to create a cleaner, more compatible playback stream and automatically falls back to Simple mode if it cannot start.
Path to the ffmpeg binary used by Enhanced mode. Use ffmpeg if it is available on PATH.
Choose how Enhanced mode encodes video. Start with Software if you are unsure, then switch to the hardware encoder that already works well on your server.
Unused Enhanced mode streams are stopped and cleaned up automatically after this many minutes.
Maximum number of Enhanced mode playbacks at the same time. Extra playbacks automatically fall back to Simple mode.

YouTube Sources

Add Source

Paste the full YouTube channel or playlist URL, for example: https://www.youtube.com/@ChannelName, https://www.youtube.com/channel/UCxxxxxx, or https://www.youtube.com/playlist?list=PLxxxxxx. A bare channel/playlist ID is also accepted for backward compatibility.
Used as the folder name inside your Jellyfin library.
Written into the source .nfo file.
